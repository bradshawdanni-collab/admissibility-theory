# Framework Maturity Spectrum

The Maturity Spectrum is a diagnostic tool. It classifies where any research program currently sits on the path from conceptual proposal to demonstrated result.

Its purpose is not to exclude work at early stages — precursory work has value. Its purpose is to make the *current stage explicit*, so that neither the author nor the reader mistakes a conceptual sketch for a demonstrated result.

---

## The Spectrum

| Stage | Description | Admissibility Status |
|---|---|---|
| **Conceptual sketch** | Vocabulary used; formal objects not defined | Not yet admissible |
| **Motivated analogy** | Formal objects referenced but not connected to claims | Not yet admissible |
| **Informal hypothesis** | Claim stated; discriminating observation not specified | Not yet admissible |
| **Formal hypothesis** | Claim stated; discriminating observation specified; terms defined | Conditionally admissible |
| **Bounded claim** | Scope conditions stated; failure conditions named | Admissible |
| **Demonstrated claim** | Predictions confirmed or refuted; chain of custody documented | Fully admissible |

---

## How to Use It

Apply the spectrum to any claim, framework, or research program by asking:

1. Are the key terms formally defined? *(If no: Conceptual sketch or Motivated analogy)*
2. Is a discriminating observation specified? *(If no: Informal hypothesis)*
3. Are scope and failure conditions stated? *(If no: Formal hypothesis)*
4. Have predictions been tested with a documented inference chain? *(If no: Bounded claim)*
5. Has the chain been independently checked? *(If no: Demonstrated claim, unreplicated)*

The output is not a judgment — it is a location. Knowing where a framework sits on the spectrum tells you what work remains, not whether the framework is worthwhile.

## Why Independent Frameworks Often Stall

Most independent research programs contain genuine insight. The common failure is not bad ideas. It is stalling at the transition between stages — specifically between:

- **Motivated analogy → Formal hypothesis**: the point at which vocabulary must become specification
- **Formal hypothesis → Bounded claim**: the point at which scope must be explicitly limited
- **Bounded claim → Demonstrated claim**: the point at which the author must allow the framework to be wrong

The third transition is the hardest. A framework that cannot be tested is one the author has not yet been willing to risk.
