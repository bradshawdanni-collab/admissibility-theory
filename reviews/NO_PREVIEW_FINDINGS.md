# No Preview Findings Rule

Do not request or circulate preliminary substantive findings before the reviewer submits the completed review.

Permitted pre-submission communication:

- access assistance;
- link repair;
- file-location clarification;
- GitHub or template mechanics;
- confirmation that no deadline applies.

Not permitted:

- asking whether the reviewer agrees;
- seeking reassurance about likely results;
- debating emerging criticisms;
- supplying intended meanings for unclear terms;
- revising doctrine in response to informal reviewer reactions;
- quoting preliminary views as endorsement.

This protects both reviewer independence and the evidentiary value of the completed review.
