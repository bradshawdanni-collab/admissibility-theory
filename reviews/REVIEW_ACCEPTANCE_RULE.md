# Review Acceptance Rule

A completed review is accepted when:

1. the reviewer and reviewed commit are identifiable;
2. the submission source is documented;
3. the full review is preserved without substantive editing;
4. the accepted file has a recorded SHA-256 digest;
5. author responses are stored separately;
6. the intake issue links to the accepted artifact and commit.

Acceptance does not require a favorable conclusion, agreement with the doctrine, or completion of every optional template field. Negative, conditional, ambiguous, and low-confidence findings remain valid evidence when their reasoning path is recorded.
