# Submission Instructions

## Preferred submission

Create one reviewer-authored Markdown file in `submissions/` named:

`YYYY-MM-DD_frank_oswald_independent_review.md`

Attach the completed independence declaration as:

`YYYY-MM-DD_frank_oswald_independence_declaration.md`

A signed PDF may be included in addition to, but not instead of, the text record.

## GitHub method

Preferred options:

1. Submit the two files through a pull request that changes only `independent_reviews/frank_oswald/submissions/`; or
2. Email the completed files to the project owner for archival upload without alteration.

The reviewer should not modify:

- frozen research materials;
- source hashes;
- manifests;
- prior review records; or
- project conclusions.

## Procedural questions

Questions about file access, broken links, naming, or missing documents may be raised during review. Substantive interpretation questions should be recorded in the review rather than answered through coaching.

## Change control

If the project owner issues a corrected package:

- assign a new release tag or package hash;
- retain the original package;
- record the reason for correction;
- ask the reviewer whether the correction requires a fresh or supplemental review.

## Completion gate

The review is complete only when the record contains:

- the reviewed release identifier;
- the independence declaration;
- the evaluation;
- source citations;
- a clear determination; and
- any disclosed conflict or limitation.
